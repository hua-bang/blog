# 可交互的信息

近几年一直在思考一个概念，

**interactive information**,

即可交互性信息。


目前并没有对这个概念有官方定义，

我个人将其理解为，

**信息是动态，双向反馈的，**

**在不同的场景形态下，**

**能展示不同的信息量。**

![图片-1](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764484206877-generated-1764484206877-yr2y307zz4k.png)

## 信息演化历史

> 欲知未来，先知过去。

《信息简史》是我很喜欢的一本书，

以时间叙事来书写信息的发展历史，

也结合本书的内容做了下简要梳理。

### 1. 前信息理论时代(远古-19世纪)

- **形态**：鼓语、烽火、信鸽
    
- **核心局限**：信息与物理载体紧密绑定,传播低效
    

### 2. 信息固化时代(文字-印刷术)

- **形态**：楔形文字、古腾堡革命
    
- **突破**: 信息永久化,可大规模复制
    
- **局限**: 信息静态,无法适应读者
    

### 3. 信息数字化萌芽(电报时代)

- **形态**：摩尔斯码、电话
    

- **突破**: 信息与物质分离,实时传输
    

- **局限**: 仍是点对点,无法群体协作
    

### 4. 信息理论诞生(香农革命)

- **形态**：1948年,信息成为可量化的数学对象
    
- **意义**: 奠定了所有现代通信的理论基础
    
- **局限**: 只关心传输效率,不关心意义和适应性
    

### 5. 信息计算时代(计算机-互联网)

- **形态**： 图灵机、个人电脑、万维网、推荐系统
    
- **突破**: 信息可编程、可检索、可分享、可推荐
    
- **局限**: 信息依然静态预设，被动个性化
    

![图片-1](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764486350810-generated-1764486350809-kuedck6s32k.png)![](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764486493129-image.png)

在信息演化过程中，

会发现**载体媒介**、**传播机制**、**传递效率**都在不同优化。

这也让我们今天获取信息变得更为容易。

但貌似信息本身的内容还是偏静态的，

即意味着有下方的局限

他自身不会蕴含或延伸其他内容。

也无法获取你这里的反馈。

但近几年 LLM 的出现，

**使得内容动态起来了，**

**并且可以互相反馈，**

**变得更可交互了。**

## **interactive information** 是什么

**个人理解**

实际上，**interactive information** 目前应该也没有确切的定义。

我姑且把它定义为**【信息是动态，双向反馈的，在不同的场景形态下，能够基于交互展示不同的信息量】**

本身从“静态存储”变成了“动态生成”

![优雅流动的交互信息](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764492106069-generated-1764492106069-ep87cv0aw5f.png)

**核心维度**

比较核心的几个维度

1. **动态性**：信息内容可根据上下文实时变化。
    
2. **双向反馈**：信息能够感知接受者反馈并调整输出
    
3. **场景适应性**：同一信息在不同场景下呈现不同的信息粒度。
    

而这些维度，实际上扩展了更多的信息量。

![图片-1](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764507332812-generated-1764507332812-1bsnbgdgk06.png)

**优点**

![](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764492046516-img_v3_02s7_dfa25f19-96a8-4e4a-8954-84220b0e884g.jpg)

- **个性化**：每个人看到界面交互可能不同。
    
- **降低认知负荷**：只展示你需要的信息数据，适配你场景的信息粒度。
    
- **双向反馈**：对于用户，可以获得更多的即时反馈。
    
- **非结构/非规则信息流动**：非结构和非规则化的信息数据，可以脱离预设的程序流程，从而流动起来。
    

**缺点和挑战**

- **用户体验层面**：用户习惯需适应、信任问题、使用成本。
    
- **技术层面**：性能要求、体验要求、一致性保证性、质量控制、设计系统等等。
    

**使用场景**

- **专题学习**：根据学习者的具体情况，动态调整内容深度与探索路径。
    
- **课程教育**：将静态教材转化为可操作的模拟环境，让学生在参数调整与实时反馈中“从做中学”。
    
- **查数提效**：通过对话让数据随思维流动，自动聚合关键指标并即时生成针对性的动态分析报告。
    
- **概念理解**：利用多维视角切换与动态可视化演示，将晦涩的抽象定义转化为可感知的直观体验。
    

## 业界的实践

近几年也确实出了一些应用

在探索比较新的人机交互。

而其中有些应用，

和 **interactive information** 有些交集。

### Gemini Dynamic Layout

**定位：**根据搜索意图实时生成交互式工具和模拟器

**典型场景：**

- 搜索"比较旅行方案" → 生成可交互的对比表
    
- 搜索"计算器" → 生成可用的计算工具
    
- 搜索"数据可视化" → 生成图表并支持参数调整
    

**例子：**用动态交互的形式来了解 Transformer

![](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764503633641-image.png)

### Anthropic Claude Artifacts

**定位**：Claude Artifacts 是 Anthropic 推出的一项功能,让 Claude 在对话中能够**直接生成可执行、可交互的内容**。

**典型场景**：快速原型验证、学习与教育、数据可视化、内容创作。

**例子**：用 Claude Artifacts 生成 AI 发展时间轴

![](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764503498160-image.png)

### Apps in ChatGPT

**定位**：将第三方应用直接嵌入 ChatGPT 对话中,通过自然语言驱动交互式界面,实现「对话即操作」

**典型场景**：能够做旅游规划、设计创作、学习教育的场景。

**例子**：在 ChatGPT 中对话，然后可以通过 Canvas 直接进行生图。

![](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764503741071-image.png)

### 阿里灵光 App

**定位**：全模态 AI 助手，将“对话”转化成“可交互应用+多模态内容”，30 秒自然语言到可用小程序。

**典型场景**：快速生成小应用，多模态创作、智能视觉理解。

![](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764506966141-image.png)

### YouWare/响指

**定位**：AI 编程社区平台,通过「氛围编程(Vibe Coding)」让非程序员也能用自然语言创建和部署 Web 应用

**典型场景**：快速原型、代码内容社区、社区内容 Remix

![](https://llm-1258926018.cos.ap-guangzhou.myqcloud.com/uploads/1764506879623-image.png)

## 未来假想

实际上，**interactive information 在不断出现**

个人对此比较兴奋，同时也有几点看法

- **信息可以适配人**：随着 形态转换， GenUI 等技术成熟，以后的信息可能会越来越适配具体的人，具体的场景。
    
- **混合 UI 交互：**虽然全文讲了较多动态交互，但现状的预设静态交互也是不可少的，两者非对立，可以混合使用。
    
- **软件方式变化**：软件从偏完全预设转向了部分动态内容生成，开发方式也需要适配。
    
- **技术基建发展：**前端交互层面，动态加载，动态渲染的基建也许有发展的空间。
    

**Information is dynamic, relatable, computable, and operates on bidirectional feedback.**  
  
信息是动态，可关联，可计算的，双向反馈的。
